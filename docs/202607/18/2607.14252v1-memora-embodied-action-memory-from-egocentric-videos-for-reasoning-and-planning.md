## 摘要
长期机器人规划不仅需要预测下一步行动，还需要记忆具身体验，使未来目标变得可解释。人类并非仅凭当前场景进行规划：他们依靠记忆中的地点、对象状态变化、先前程序以及通过重复行动揭示的规律。我们将具身动作记忆（EAM）定义为形成、维持并利用这种经验作为持久记忆状态以指导后续决策的能力。MEMORA 通过形成-巩固-检索生命周期和四种类型存储实现了 EAM：环境记忆、实体记忆、活动记忆和推断知识。在线编辑在接收到新观察时维护对象身份和状态历史；离线巩固将重复经验抽象为可复用程序和参与者特定的规律。MEMORA-Bench 在 18 名参与者的 45 小时 EPIC-KITCHENS-100 扩展视频上，通过基于记忆的规划（包括未见目标）和互补的记忆评估任务评估了这一生命周期。在四种开放权重语言模型中，完整的 MEMORA（结合编辑、类型存储和巩固）在评估的记忆条件下取得了最强的综合结果。与最强控制基线相比，它将记忆评估准确性提高了最多 20.5 个百分点，并将分布外机器人接地规划分数相对提高了最多 16.6%。一项定性的双任务机器人部署研究进一步说明了基于记忆的语言规划如何与下游控制交互，而整体结果表明，可编辑、可巩固的记忆可以为机器人规划提供记忆上下文。项目页面：https://yuzihaowashu.github.io/MEMORA/

## Abstract
Long-horizon robot planning requires more than predicting what actions will do next; it also requires memory of the embodied experience that makes future goals interpretable. People do not plan from the present scene alone: they draw on remembered places, object-state changes, prior procedures, and regularities revealed through repeated action. We formulate Embodied Action Memory (EAM) as the capability to form, maintain, and use such experience as a persistent memory state for later decisions. MEMORA realizes EAM with a formation-consolidation-retrieval lifecycle and four typed stores: Environment Memory, Entity Memory, Activity Memory, and Inferred Knowledge. Online editing maintains object identities and state histories as new observations arrive; offline consolidation abstracts repeated experience into reusable procedures and participant-specific regularities. MEMORA-Bench evaluates this lifecycle on 45 hours of EPIC-KITCHENS-100 extension video across 18 participants through memory-grounded planning, including previously unseen goals, and a complementary memory-assessment task. Across four open-weight language models, full MEMORA--combining editing, typed stores, and consolidation--achieves the strongest aggregate results among the evaluated memory conditions. It improves memory-assessment accuracy by up to 20.5 points over the strongest controlled baseline and improves out-of-distribution Robot-Grounded Plan score by up to 16.6% relative. A qualitative two-task robot deployment study further illustrates how memory-grounded language plans can interface with downstream control, while the overall results show that editable, consolidated memory can supply remembered context for robot planning. ## 2. 论文提出的方法论

### 核心思想
- 将具身经验组织成四类类型化存储，遵循“形成→巩固→检索”的生命周期。
- 在线编辑：随着新观察到达，通过 `ADD`、`UPDATE`、`DELETE`、`NOOP` 操作维护实体（对象）的身份和状态历史。
- 离线巩固：在完成视频处理后，将重复经验抽象为可重用的程序、习惯和偏好（即推断知识）。
- 类型化检索：为规划者提供分离的过程性证据（如何做）和物理接地证据（哪个对象、在哪里、什么属性）。

### 关键技术细节
- **四个记忆存储**：
  - *环境记忆*：空间布局、命名区域、空间关系。
  - *实体记忆*：对象记录（视觉属性、位置、当前状态、状态历史），随时间保持对象身份。
  - *活动记忆*：按时间排序的动作记录（每10秒片段摘要、动作分解）。
  - *推断知识*：通过离线巩固产生的习惯、偏好、流程模式，链接到支持性证据。
- **在线记忆形成**：使用多模态模型（Segment Encoder）将每10秒视频片段转换为结构化观察，然后通过 Memory Editor 执行编辑操作更新实体记忆，环境记忆和活动记忆采用确定性更新。
- **离线巩固**：在所有视频处理后，对实体和活动证据进行分析，提取跨会话的规律，形成推断知识。
- **检索与规划**：采用 ReAct 循环，提供类型感知搜索工具（对象、活动、环境、模式等），以及时间受限推理工具（`get_state_at_time`、`get_object_history`）。规划器首先检索程序性证据（习惯、偏好），然后接地到具体实体和地点。

### 公式或算法流程（文字说明）
- 记忆形成函数：$M_t = F(M_{t-1}, o_t)$，其中 $F$ 由编码（$\phi$) 和编辑（EDIT）组成：$z_t = \phi(v_t, c_{t-1})$，$M_t = \text{EDIT}(M_{t-1}, z_t)$。

## 6. 论文的主要结论与发现

- **全面最优**：完整 MEMORA 在所有 backbone 和任务类型上取得最佳综合 RGP 分数，尤其在分布外 Generalize 任务上优势更大（相对提升最多 +16.6%）。
- **离线巩固的关键作用**：对于跨会话记忆（偏好、习惯、流程）和分布外规划，离线巩固带来显著提升。在 EAM-QA 上，巩固贡献 +14.4 个绝对百分点；在 Generalize 规划上，帮助最大的两个推理 backbone 提升约 +0.025 至 +0.022 RGP。
- **在线编辑的作用**：编辑减少了约 18× 的实体记录冗余（图2），提高了记忆质量。将 Memory Editor 从 Qwen3-30B 降为 Qwen3-14B 导致 EAM-QA 准确率下降 5.8 点，且“信息不可用”选项选择率从 42.8% 升至 62.3%。
- **类型化存储的优势**：与扁平文本或关系图相比，四种类型化存储分离了程序和接地证据，使规划器能分别检索习惯/偏好和实体/位置，尤其在 Generalize 任务中效果显著。
- **感知模态贡献较小**：将多模态感知（音视频）替换为纯视觉仅损失 2.7 点 EAM-QA 准确率，表明记忆编辑器的容量比感知模态更关键。

## 7. 优点

- **系统性生命周期设计**：首次将记忆的形成、编辑、巩固、检索完整生命周期应用于具身机器人规划，结合认知科学的多存储和巩固理论。
- **实际视频基准**：使用真实人类自我中心视频（EPIC-KITCHENS-100）而非合成数据，任务和问题基于真实动作注释，具有生态效度。
- **公平的消融设计**：通过多次条件阶梯（7 级）和三个消融轴，清晰隔离了编辑、巩固、感知、存储类型各自贡献。
- **开放权重模型**：整个管道完全基于开放权重模型（Qwen、Gemma），可复现性好，不含闭源模型（闭源模型仅作为可选替代对比）。
- **交叉验证**：在4种不同架构（MoE vs dense）、不同参数量的 backbone 上验证，结论一致。

## 8. 不足与局限

- **领域局限**：实验仅在厨房场景（EPIC-KITCHENS-100）中，最多 15 个会话，无法验证数月的漂移、季节性变化或稀有事件。
- **管道依赖性**：记忆质量受感知和编辑模型影响大；感知错误可能传播到巩固知识中，除非被后续观测纠正。
- **机器人部署有限**：仅定性演示两个任务，使用固定规则控制器，未实现完整闭环自主（导航、变形操作、在线感知更新、失败恢复等）。
- **隐私考量**：系统构建的是参与者特定的记忆，实际部署需考虑同意、审查、访问控制和记忆共享限制，论文仅提及未深入解决。
- **泛化任务数量较少**：Generalize 任务仅 153 个，可能不足以覆盖所有转移/组合场景。
- **对象接地指标缺陷**：论文指出 token 重叠指标（如 KBGround、ObjGT-F1）不能公平反映 MEMORA 的优势，因为 MEMORA 会改写表述导致得分降低，但未提供更优的接地评估。
