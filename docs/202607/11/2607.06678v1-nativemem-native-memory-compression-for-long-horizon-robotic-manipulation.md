## 摘要
预训练视觉-语言-动作（VLA）模型如何在不牺牲效率的情况下，以高频更新保留长时域视觉历史？现有方法依赖外部记忆管理，这限制了记忆跨度或预训练策略的反应性。为此，我们提出 NativeMEM，一种具有长期和实时更新记忆的 VLA 策略。其核心是一种高效的记忆编码方案——本机内存压缩，它重新利用 VLA 自身的视觉编码器，将每个相机视图的历史帧压缩为单个 token。这些记忆 token 被附加到输入序列中，使预训练 VLA 能够关注长时域历史，且延迟开销可忽略不计，无需外部规划器或重新初始化的记忆模块。为使记忆 token 与预训练策略对齐，我们首先在冻结的 VLA 监督下，在记忆需求数据上开发一个通用的记忆分词器，然后解冻 VLA 进行任务特定微调。NativeMEM 始终优于先前方法，在仿真中将成功率从 32.4% 提升至 84.0%，在真实机器人上最高达 98.7%，同时保持低推理延迟和 GPU 内存使用。值得注意的是，NativeMEM 展现了高数据效率，仅用 20% 的训练数据就取得了与先前技术相当的结果。

## Abstract
How can pretrained Vision-Language-Action (VLA) models retain long-horizon visual histories with high-frequency updates without sacrificing efficiency? Existing approaches rely on external memory management, which restrains either the memory horizon or the reactiveness of pretrained policies. To this end, we present NativeMEM, a VLA policy that features long-term and real-time updated memory. At its core is an efficient memory encoding scheme, Native Memory Compression, which repurposes the VLA's own vision encoder to compress each historical frame from each camera view into a single token. Appended to the input sequence, these memory tokens enable the pretrained VLA to attend over long-term history with negligible latency overhead, requiring neither an external planner nor a freshly initialized memory module. To align the memory tokens with the pretrained policy, we first develop a generic memory tokenizer under the supervision of a frozen VLA on memory-demanding data, and then unfreeze the VLA for task-specific fine-tuning. NativeMEM consistently outperforms prior methods, boosting success rates from 32.4% to 84.0% in simulation and up to 98.7% on real robots, while maintaining low inference latency and GPU memory usage. Notably, NativeMEM exhibits high data efficiency by achieving competitive results with prior arts using only 20% of the training data.

## 1. 核心问题与整体含义（研究动机和背景）
- **问题**：预训练的视觉-语言-动作（VLA）模型大多仅基于当前观测做出反应，缺乏长期历史记忆能力，无法胜任需要追踪任务进展、交互计数、遮挡状态等记忆依赖的长时域操作。
- **现有方案的局限**：
  - **外部VLM驱动记忆**：将历史转换为文本规划，但文本难以捕捉细粒度视觉细节，且需要昂贵标注。
  - **外部记忆模块**：引入新模块（如循环状态、检索库）会增加架构复杂度，且与预训练策略不兼容，容易破坏原有性能。
  - **压缩困境**：压缩不足则历史开销过大；压缩过度则丢失动作关键信息。
- **核心动机**：如何用极低的计算代价，在保留预训练兼容性的前提下，为VLA注入长期、细粒度的视觉历史记忆。

## 2. 方法论：核心思想、关键技术细节与流程
- **核心思想 – 原生记忆压缩（Native Memory Compression）**：
  - 复用VLA自带的视觉编码器，将每个相机视图的历史帧压缩为**单个记忆token**（极致压缩，每帧每视角仅1个token）。
  - 这些“原生记忆token”直接附加到VLA原有的输入序列中，无需新增外部模块，利用现有注意力机制访问长时历史。
- **记忆分词器设计**：
  - 从VLA视觉编码器初始化记忆编码器，加入可学习的记忆查询token `q_mem`，对视觉补丁序列进行交叉注意力，得到摘要token `ˆq_vτ`。
  - 通过线性投影映射到VLA token维度，得到记忆token `m_vτ`。
  - 记忆序列由可学习的起头token `b_mem` 和各帧记忆token拼接而成：`M_t = [b_mem, {m_vτ | τ∈I_t, v=1..V}]`。
  - 记忆序列直接拼接到当前观测token和指令语言token之后，形成完整输入。
- **两阶段训练流程**：
  - **第一阶段：记忆对齐训练**
    - 冻结预训练VLA，仅训练记忆分支。
    - 使用VLA原始动作损失优化记忆分词器，使记忆token与预训练token空间对齐，且直接动作相关（无重建损失）。
    - 训练数据混合常规操作和海量记忆需求任务（仿真+真实），并在每个训练步使用首帧和近期历史窗口作为记忆索引。
  - **第二阶段：任务特定微调**
    - 使用第一阶段训练好的分词器离线预处理所有历史帧，缓存记忆token。
    - 冻结分词器，解冻VLA backbone、动作头和记忆投影层，在少量目标任务演示上微调。
    - 保持标准VLA训练流程，仅增加记忆token输入。
- **实时推理架构**：记忆分词器独立于策略异步运行，高频更新记忆队列；策略需要动作时直接读取最新记忆队列拼接，推理延迟极低。

## 7. 优点：方法与实验的亮点
- **架构简洁且兼容**：完全复用预训练VLA视觉编码器，不引入额外记忆模块，无外部规划器，贴合现有VLA部署流程。
- **极致压缩策略**：每帧每视角仅1个token的压缩率，在保持细粒度历史的同时大幅降低序列长度。
- **动作监督对齐**：通过原生动作损失训练记忆分词器，确保记忆token对动作预测有用，而非单纯视觉重建。
- **两阶段分离设计**：先通用对齐，后任务微调，保护了预训练先验，实现高数据效率。
- **强大泛化能力**：记忆分词器可在未见（unseen）记忆任务上直接泛化，证明学到的压缩策略具有任务迁移性。
- **实验全面扎实**：同时在仿真和真实世界任务上验证，覆盖多种记忆需求类别，消融合理，baseline公正复现。
