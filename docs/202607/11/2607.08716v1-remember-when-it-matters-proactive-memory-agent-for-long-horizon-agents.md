## 摘要
在长程任务中，与决策相关的状态往往分散在不断扩展的轨迹中，而行动智能体需要将其浮现出来并据此行动。随着轨迹增长，任务要求、环境事实、先前的尝试、诊断和未完成的子目标可能会被掩埋在上下文窗口内或超出其范围，从而在需要时无法影响决策。我们将这种失效模式称为“行为状态衰减”。我们将记忆视为一种主动干预机制，而非被动检索。一个独立的记忆智能体与未经修改的行动智能体并行运行，根据最近的轨迹更新结构化记忆库，并决定是注入基于记忆的提醒还是保持沉默。该模块可与前沿行动智能体和现有智能体框架即插即用。在Terminal-Bench 2.0和τ²-Bench上，它为较弱和较强的行动智能体都提高了pass@1，在Terminal-Bench上提升8.3个百分点，在τ²-Bench上提升6.8个百分点。消融实验表明，选择性干预优于被动暴露记忆库、始终开启的注入、仅顾问指导以及通用检索。作为迈向开放权重记忆策略的早期步骤，我们使用SFT和GRPO在SETA上训练Qwen3.5-27B，提高了验证奖励，并实现了向Terminal-Bench的部分迁移。

## Abstract
In long-horizon tasks, decision-relevant state is often scattered across an expanding trajectory, while the action agent must surface it and act. As trajectories grow, task requirements, environment facts, prior attempts, diagnoses, and open subgoals can be buried in the context window or pushed beyond it, failing to influence decisions when needed. We call this failure mode "behavioral state decay". We study memory as an active intervention mechanism rather than passive retrieval. A separate memory agent runs alongside an unmodified action agent, updating a structured memory bank from the recent trajectory and deciding whether to inject a memory-grounded reminder or remain silent. The module is plug-and-play with frontier action agents and existing agent harnesses. Across Terminal-Bench 2.0 and $τ^2$-Bench, it improves pass@1 for both weaker and stronger action agents, with gains of +8.3 pp on Terminal-Bench and +6.8 pp on $τ^2$-Bench. Ablations show that selective intervention outperforms passive bank exposure, always-on injection, advisor-only guidance, and general retrieval. As an early step toward open-weight memory policies, we train Qwen3.5-27B on SETA using SFT and GRPO, improving validation reward and achieving partial transfer to Terminal-Bench.

## 2. 论文提出的方法论：核心思想、关键技术细节、算法流程

- **核心思想**：引入一个独立的**记忆智能体**（memory agent），与原来的动作智能体（action agent）并行运行，无需修改后者。记忆智能体的任务是：
  1. 从最近的轨迹中更新一个结构化的记忆库；
  2. 根据当前状态判断是否需要向动作智能体的下一次调用注入基于记忆的提醒（或以空操作保持沉默）。
- **关键技术细节**：
  - **记忆库结构**：包含三个部分：
    - 私有状态字段 `s_t`（用于记忆智能体自身追踪进度，不暴露给动作智能体）；
    - 知识记忆集合 `K_t`（稳定的事实，如任务要求、环境属性、文件路径等）；
    - 程序记忆集合 `P_t`（尝试与结果，如失败的命令、成功的修复、诊断信号等）。
  - **两阶段工作流**：
    - **阶段1（记忆管理）**：根据任务描述、最近轨迹窗口和当前记忆库，通过工具调用（`memory_update_status`、`memory_save_knowledge`、`memory_save_procedural`、`memory_delete`）更新记忆库。
    - **阶段2（干预选择）**：基于更新后的记忆库和最近轨迹，决定输出一个以记忆为依据的提醒 `r_t`，或是显式的空干预 `∅`。如果输出提醒，则将其作为临时上下文注入到下一次动作智能体的调用中。
  - **触发机制**：按固定步数间隔调用记忆智能体（第一步和之后每 N 步），以便研究干预策略本身的效果。

## 6. 论文的主要结论与发现

- 在长程任务中，主动记忆干预能有效缓解行为状态衰减，显著提升动作智能体的任务完成率。
- 在 Terminal-Bench 2.0 上，使用 Claude Opus 4.6 作为记忆智能体，Sonnet 4.5 动作智能体提升 +8.3 pp，Opus 4.6 动作智能体提升 +2.4 pp；在 τ²-Bench 上，对应提升分别为 +6.8 pp 和 +2.5 pp。
- 消融实验表明，仅维持记忆库不够（全库上下文增益有限），始终注入提醒不如选择性干预平衡，无记忆库的顾问式指导不够稳定，通用存储检索也无法替代主动干预的校准作用。
- 定性地，记忆干预常在动作智能体即将忽略关键要求、环境事实、失败诊断或程序顺序时发挥效用，提醒精确、及时。
- 初步训练实验显示，记忆干预策略可以被蒸馏到开放权重模型，并在经过 GRPO 优化后能够迁移到新任务。
